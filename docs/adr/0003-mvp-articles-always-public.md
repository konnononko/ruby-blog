# ADR 0003: MVP では記事に下書き・公開フラグを置かず、投稿済み記事はゲストも含め一覧・詳細で閲覧可能とする

日付: 2026-05-02

## ステータス

採用

## コンテキスト

ブログの最初のリリースで、認証・投稿・コメント・閲覧の一連の流れを先に固めたい。下書き・予約投稿・非公開の扱いはスキーマと画面の複雑さを増やす。

## 決定

- MVP では `Article` に「公開／下書き」や「公開日時」といった列を持たない。掲載を止める操作は、作者による削除（と必要なら将来の非表示機能）で扱う想定とする。
- 記事の一覧と詳細は、未ログインのゲストも閲覧できる。新規・編集・削除はログイン必須で、編集・削除は作者に限定する（別 ADR・計画書の認可と整合）。

## 結果

- 良い点: 実装とテストの範囲が小さく、公開の意味が単純で説明しやすい。
- 悪い点: 誤投稿の取り消しは削除頼みになる。下書きや限定公開が必要になったら、マイグレーション・画面・認可の見直しが必要になる。
