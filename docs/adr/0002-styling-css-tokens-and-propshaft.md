# ADR 0002: フロントは Tailwind 等の UI フレームワークに依存せず、CSS 変数と素の CSS、Propshaft を用いる

日付: 2026-05-02

## ステータス

採用

## コンテキスト

ブログの見た目を揃え、かつビルドや依存の重さを MVP の段階で抑えたい。Rails 8 では Propshaft がアセット配信の前提となっている。

## 決定

- スタイリングは `app/assets/stylesheets/application.css` を中心に、`:root` の CSS 変数（色・余白・影・角丸・ガラス用など）でトークンを定義し、ナビ・パネル・ボタン・フォーム等に素の CSS でクラスを当てる。
- Tailwind CSS、Bootstrap、大型のコンポーネント UI フレームワークは MVP では導入しない（必須としない）。
- アセットは Propshaft を用いる（プロジェクトの現状に合わせる）。

## 結果

- 良い点: 依存と学習コストが小さい。デザイントークンを一箇所に集めやすい。生成・ビルドパイプラインを複雑にしにくい。
- 悪い点: 画面数が増えるとユーティリティ集めの工数は自前になる。今後、デザイン体制や移植性の要件が変わったら、CSS フレームワークの導入を再検討してよい。
